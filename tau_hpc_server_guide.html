1. עקרונות עבודה נכונים עם שרת חישובים

שרת הכניסה, למשל powerslurm-login3, מיועד לעריכת קבצים, שליחת jobs, בדיקת תוצאות וניהול עבודה. חישובים כבדים צריך לשלוח לתור דרך sbatch.

!
כלל זהב: לא מריצים MATLAB כבד, סימולציות, לולאות גדולות או עומס CPU ישירות על שרת הכניסה. משתמשים ב־SLURM.

מה עושים על login node?

מתחברים, מעלים קבצים, עורכים סקריפטים קצרים, שולחים jobs, בודקים logs, מורידים תוצאות.

מה עושים דרך SLURM?

MATLAB, MATPOWER, Python כבד, סימולציות, חישובי מפות, ריצות ארוכות, חישוב מקבילי.

2. התחברות מהבית דרך VPN

כאשר עובדים מהבית, בדרך כלל צריך להתחבר קודם ל־VPN של האוניברסיטה. אחרי שה־VPN מחובר, ניתן לבצע SSH ו־SFTP לשרת.

7. בסיס SLURM: איך שולחים עבודה לשרת

ב־SLURM יוצרים קובץ .sh עם דרישות משאבים, ואז שולחים אותו עם sbatch.

בדיקת partitions זמינים

בדיקת תורים זמינים

sinfo -p power-general-shared-pool -o "%P %c %m %D %l %N"

תבנית SLURM בסיסית

run_job.sh
#!/bin/bash
#SBATCH --job-name=my_job
#SBATCH --partition=power-general-shared-pool
#SBATCH --account=amir-users_v2
#SBATCH --qos=public
#SBATCH --output=my_job_%j.out
#SBATCH --error=my_job_%j.err
#SBATCH --time=01:00:00
#SBATCH --cpus-per-task=1
#SBATCH --mem=4G

cd $SLURM_SUBMIT_DIR

/usr/local.cc/bin/matlab -batch "my_matlab_script"

9. בדיקת Parallel MATLAB / parpool

לפני ריצות מקביליות, חשוב לבדוק שהשרת מסוגל לפתוח workers לפי מספר ה־CPUs שביקשת מ־SLURM.

סקריפט MATLAB לבדיקת parpool

test_parallel_hpc.m
clear;
clc;

fprintf('============================================================\n');
fprintf('Parallel MATLAB smoke test on HPC\n');
fprintf('============================================================\n');

fprintf('MATLAB version: %s\n', version);
fprintf('Working directory:\n%s\n', pwd);

nWorkers = str2double(getenv('SLURM_CPUS_PER_TASK'));
if isnan(nWorkers) || nWorkers < 1
    nWorkers = 4;
end

fprintf('Requested workers from SLURM_CPUS_PER_TASK: %d\n', nWorkers);

p = gcp('nocreate');
if ~isempty(p)
    delete(p);
end

fprintf('Opening parallel pool with %d workers...\n', nWorkers);
p = parpool('Processes', nWorkers);
fprintf('Parallel pool opened. NumWorkers = %d\n', p.NumWorkers);

N = 24;
out = zeros(N, 1);

tic;
parfor k = 1:N
    out(k) = sum(sin((1:200000) * k * 1e-7));
end
tParallel = toc;

fprintf('parfor completed.\n');
fprintf('Elapsed time: %.3f seconds\n', tParallel);
fprintf('Output checksum: %.12f\n', sum(out));

delete(gcp('nocreate'));

fprintf('Parallel MATLAB smoke test completed successfully.\n');

שליחת בדיקת 32 workers

10. הרצת חישוב כבד בצורה נכונה

לעבודה כבדה מומלץ להשתמש ב־wrapper MATLAB ובקובץ SLURM נקי. כך נמנעים מבעיות גרשיים בפקודת matlab -batch.

עקרון מומלץ

קובץ driver

הסקריפט הראשי של הפרויקט, למשל run_stability_analysis_professional_parallel.m.

קובץ wrapper

מוסיף paths, בודק שהקבצים קיימים, ואז קורא ל־driver.

קובץ SLURM

מבקש CPUs, זיכרון וזמן, ואז מריץ את ה־wrapper.

Wrapper MATLAB לדוגמה

run_main_full_hpc_wrapper.m
clear;
clc;

fprintf('============================================================\n');
fprintf('HPC FULL main wrapper started\n');
fprintf('============================================================\n');

project_dir = pwd;
matpower_path = '/a/home/cc/students/enginer/yossefm1/software/matpower8.1';

addpath(genpath(matpower_path));
addpath(genpath(project_dir));

fprintf('Project dir:\n%s\n', project_dir);
fprintf('MATPOWER path:\n%s\n', matpower_path);

fprintf('Driver path:\n%s\n', which('run_stability_analysis_professional_parallel'));
fprintf('compute_H path:\n%s\n', which('compute_H'));

if isempty(which('run_stability_analysis_professional_parallel'))
    error('Main driver script was not found.');
end

run_stability_analysis_professional_parallel;

fprintf('============================================================\n');
fprintf('HPC FULL main wrapper completed successfully\n');
fprintf('============================================================\n');

SLURM לריצה כבדה

run_main_full_32.sh
#!/bin/bash
#SBATCH --job-name=grid_full32
#SBATCH --partition=power-general-shared-pool
#SBATCH --account=amir-users_v2
#SBATCH --qos=public
#SBATCH --output=grid_full32_%j.out
#SBATCH --error=grid_full32_%j.err
#SBATCH --time=24:00:00
#SBATCH --cpus-per-task=32
#SBATCH --mem=96G

set -x

echo "Job started at: $(date)"
echo "Host: $(hostname)"
echo "SLURM_JOB_ID: $SLURM_JOB_ID"
echo "SLURM_CPUS_PER_TASK: $SLURM_CPUS_PER_TASK"
echo "SLURM_SUBMIT_DIR: $SLURM_SUBMIT_DIR"

cd $SLURM_SUBMIT_DIR

pwd
ls -lh run_main_full_hpc_wrapper.m
ls -lh run_stability_analysis_professional_parallel.m

echo "Starting MATLAB full run..."
/usr/local.cc/bin/matlab -batch "run_main_full_hpc_wrapper"

echo "MATLAB exit code: $?"
echo "Job ended at: $(date)"
בדיקה ושליחה

11. מעקב וניהול jobs

לאחר שליחת job ניתן לצאת מהשרת. ה־job ממשיך לרוץ. כדי לעקוב, נכנסים שוב ובודקים את התור ואת קבצי הפלט.

בדיקת jobs פעילים
squeue --me
בדיקת מצב job שסיים
sacct -j JOB_ID --format=JobID,JobName,State,ExitCode,Elapsed,ReqCPUS,ReqMem,NodeList%30
מעקב חי אחרי פלט
tail -f $(ls -t grid_full32_*.out | head -n 1)

כדי לצאת מ־tail -f, לחץ Ctrl + C.

עצירת job
scancel JOB_ID

13. תקלות נפוצות ופתרונות

1. job הסתיים מהר מדי וה־out ריק

בדוק את סטטוס ה־job עם sacct. אם ExitCode הוא 0 אבל אין פלט, ייתכן שהפקודה ב־SLURM לא הריצה את הסקריפט הנכון.

בדיקת job שסיים
sacct -j JOB_ID --format=JobID,JobName,State,ExitCode,Elapsed,ReqCPUS,ReqMem,NodeList%30

2. בעיית גרשיים ב־matlab -batch

אם מופיעה שגיאה כמו unexpected EOF while looking for matching ', השתמש ב־wrapper MATLAB במקום פקודת -batch ארוכה.

3. בדיקת נתיבים של פונקציות MATLAB

בדיקת which ב־MATLAB
/usr/local.cc/bin/matlab -batch "which runpf -all; which case30 -all; which mpoption -all"

4. אזהרת .profile בכניסה

אם מופיעה שגיאה בכניסה כמו:
